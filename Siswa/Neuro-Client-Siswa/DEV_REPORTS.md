# 🚀 PANDAI Neuro-Client Development Reports

## [v26.0] Phase: Neuro-Pro Max Performance (QUANTUM & SILK)
**Status**: Completed 🐆
**Goal**: Zero-Lag UI and ultra-responsive re-detection.

### 1. Issue: 100% CPU Usage & Dragging/Trailing Lag
- **Root Cause**: The `DecisionEngine` was calling `get_available_cameras()` every 100ms. This function performed a blocking hardware scan of 4 camera ports using `cv2.VideoCapture`. On Windows, each scan takes ~0.5s, causing an infinite loop of blocking calls.
- **Solution**: 
    - Implemented **Camera Caching** in `VisionEngine.py`. Hardware scans now only occur once every 10 seconds.
    - Reduced MQTT telemetry frequency to **5 FPS** for stable metrics to reduce browser (LMS) overhead.
    - Implemented **State-Change Bypass**: Critical updates (Vision Lost/Recovered) still send messages instantly, ignoring the throttle.
- **Impact**: CPU usage dropped significantly. UI dragging and browser scrolling are now smooth.

### 2. Issue: Slow Re-detection after Vision Loss
- **Root Cause**: High CPU load caused MediaPipe inference to slow down, creating a backlog in the thread queue.
- **Solution**:
    - Implemented a **30 FPS Limiter** in `VisionEngine._process_loop` to prevent CPU exhaustion.
    - Increased internal `DecisionEngine` monitor frequency to **20 FPS** (0.05s) for tighter sensory polling.
    - Implemented a **0.2s Debounce** to prevent sensory flickering in edge cases.
- **Impact**: Re-detection is now perceived as instant (~100ms) once the face enters the frame.

### 3. UI/UX: Sprite Stretched Aura
- **Optimization**: Replaced line-by-line drawing loops with **Linear Sprite Stretching**.
- **Impact**: Gradient rendering time reduced from ~15ms to <1ms.

## [v26.1] Architectural Guardrails: Hardware Scalability
**Requirement Standard**: 8GB+ RAM (Mandatory).

### 🖥️ Hardware Compatibility Matrix
Pengembangan fitur selanjutnya WAJIB menyesuaikan dengan profil GPU berikut:

1.  **Low GPU (Integrated Graphics / Intel HD)**:
    - **Mode**: *CPU-Dominant Optimization*.
    - **Fitur**: Wajib menggunakan `Frame Skipping` dan limitasi UI drawing. Jangan menambah AI model yang berjalan secara bersamaan di GPU.
2.  **Medium GPU (GTX/RTX 50-series or entry level)**:
    - **Mode**: *Balanced Flow*.
    - **Fitur**: Standar Vision Engine v26.0 (30 FPS) dengan rendering gradien sutra.
3.  **High GPU (RTX 3070 / 4070+)**:
    - **Mode**: *Prestige High-Fidelity*.
    - **Fitur**: Dapat diaktifkan untuk `Face Mesh Refinement` (High Contrast) dan model emosi yang lebih kompleks tanpa membebani Main Thread.

### 🛡️ Development Golden Rules
- **RAM Efficiency**: Jaga konsumsi memori di bawah 512MB (Python side) agar sisa RAM 8GB dapat digunakan secara leluasa oleh Browser/LMS.
- **Main Thread Guard**: APAPUN hardware-nya, dilarang menempatkan I/O atau Drawing loop berat di Main Thread Tkinter. Gunakan *Deferred Rendering* dan *Sprite Stretching*.
- **Async Priority**: Seluruh komunikasi (MQTT/Serial) harus berada di thread terpisah agar tidak menyebabkan "Micro-stutter" pada UI.
